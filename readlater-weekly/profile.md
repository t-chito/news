# 関心プロフィール

このファイルは「あとで読む」に保存した記事から読み取れる、現時点の関心と知識領域の地図である。時系列の記録ではなく、関心の構造を一枚で見渡すために随時書き換える。

## 中心にある関心: AIコーディングエージェントの運用設計

最も比重が大きい領域。Claude Code や Codex といったエージェントに「作らせること」はすでに前提になっていて、関心は生成の前後をどう設計するかに移っている。具体的には次の三つに分かれる。

- 絞り込みと検証。大量に出力されたものから使えるものをどう選び、別モデルや専任のレビュー役にどう批判させるか。生成と検証を別の役割に分ける設計に繰り返し触れている。
- 知識とルールの注入。プロジェクト固有の流儀をスキルや AGENTS.md、hook に切り出して継続更新する手法。「Claude がすでに知っていることは書かない」という抽出方針への共感がある。
- 権限とコンテキストの制御。permission や hook を信用しすぎず、IAM やサンドボックスで守るべきだという議論まで踏み込んでいる。

ここから「トークン経済とコンテキスト効率」が枝分かれしている。消費の主因を会話本文ではなく設定・ルール・探索的読み込みに見て、入出力・記憶・ワークフローの三層で削る発想に関心がある。圧縮率の数字を鵜呑みにせず再測定を求める慎重さも併せ持つ。

## 隣接する関心: AIに知識をどう渡すか

エージェント運用と地続きで、AIに知識を構造化して渡す設計に関心が広がっている。Obsidian や Notion を使った LLM Wiki、ナレッジグラフ、PageIndex のような推論ベース RAG、ドキュメント整合性の静的検証（contextlint）まで、道具はばらばらだが「モデルではなく知識の渡し方で賢さが決まる」という一点で束ねて読んでいる。Karpathy の LLM Wiki を起点にした実践記事を複数追っている点に、この関心の現在地が表れている。

## 開発プロセスへの関心: 仕様駆動から意図駆動へ

仕様駆動開発(SDD)を追ううちに、仕様書に実装詳細が混入して二重管理になる問題と、その反作用としての意図駆動開発(IDD)へと関心が分岐した。プロンプトの再現性を実測でチューニングする手法や、AIが失敗を観測できる足場を人間が設計するという考え方に強く反応している。「AIの出力を信頼するのではなく検証可能にする」という姿勢が、エージェント運用の関心と通底している。

## 基礎を固める関心: アーキテクチャ・設計原則・テスト

流行の AI 活用と並行して、AIに依存しない地力を固める古典的な題材を厚く拾っている。クリーンアーキテクチャや DDD、レイヤード構造の進化史、UML の盛衰、DRY/KISS/YAGNI といった設計原則を「変更を安全かつ容易にするため」という一本の軸で捉えている。テストについても、正しさの確認ではなく品質を構造として設計し、テストコードを「生きた仕様書」として人間にもAIにも渡す資産にする発想に関心がある。設計思想を知っていることが、AIの出力を評価する物差しになるという認識がここでも繰り返し現れる。

## フロントエンドへの関心: 制約を先に設計する

モダンCSS(Google の Modern Web Guidance、border-shape)、Critical Rendering Path、キャッシュ階層、ディレクトリ構成、コンポーネント設計論を追っている。「人間が制約を先に設計し、ブラウザや AI に判断を委ねる」という軸で読んでいる点が、エージェント運用やデザイン生成の関心と響き合う。計測を伴わない先回り最適化を戒める姿勢も一貫している。

## 質への関心: AIスロップと検証

AIで量を増やすこと自体には価値を置かず、人間が判断・検証できる形まで質を高めることに価値を置く。網羅性そのものは否定せず、バージョン固定や実機・ソース照合といった根拠の明示を伴うかどうかで質を見分ける。この「量より検証」の姿勢は、エージェント運用の絞り込み・検証への関心と同じ根を持つ。

## 表現とデザインへの関心

生成AIでスライドや資料を作る際、デザインの土台を事前に用意して出力の振れ幅を抑える発想に関心がある。一方で、ストーリー構成や視線誘導という伝達設計は人間の領分に残るという認識も持っている。

## 周縁の関心

点在しているが続いている領域。資格・学習法（統計検定、AI を使った学習）、セキュリティの基礎（踏み台サーバー、CVSS の読み方）、運用基盤（デプロイ高速化、ログ設計）、ローカルで動く生成AI（音楽生成、音声合成）、予測市場と金融の基礎、そしてAIと雇用をめぐる議論（AIウォッシング論）。中心の関心ほど深掘りしていないが、技術の周辺にある社会的・経済的な論点にも目を配っている。

## まだ拾えていない視点

保存はソフトウェア開発の作り手側の関心にほぼ集中している。プロダクトマネジメント、ユーザーリサーチ、ビジネスやマーケティングの視点はほとんど入っていない。AI活用についても、倫理・規制・コスト構造といったマクロな論点は雇用論を除いて手薄である。

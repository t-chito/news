まとめ

今週の最大の出来事は、先週ローンチしたばかりの Claude Fable 5 / Mythos 5 が米政府の輸出管理指令を受けて全面停止になったこと（6/13）。Claude Code を含む全プロダクトの新規セッションは Opus 4.8 など他モデルにフォールバックする運用に戻り、Cognition も Devin から Fable 5 を外した。プロダクト面では Claude Code が Claude Design との双方向同期（/design-sync）を投入し、Managed Agents がスケジュール実行・Vault 認証情報・各社サンドボックス対応で本番投入に近づいた。Anthropic は「Claude Code が実際に何に使われているか」を追う経済インデックスを公開。Claude Code 本体は 2.1.176〜2.1.181 を週内で連続リリースした。他陣営では OpenAI Codex がレートリミットのリセット貯蓄・Computer Use・プラグインマーケットを、xAI が Grok Build のプラグインマーケットを出している。

Claude Code

Fable 5 が全面停止——コーディングエージェントが回すモデルが Opus 4.8 に逆戻り

先週「今週最大のニュース」として取り上げた Claude Fable 5 が、わずか数日で使えなくなった。6 月 13 日、Anthropic は「米政府が国家安全保障上の権限を根拠に、Fable 5 および Mythos 5 への外国籍ユーザー（米国内外を問わず）によるアクセスをすべて停止する輸出管理指令を出した」と公表。これを受けて Claude Devs は「米政府の指令の結果として、全ユーザー向けに Claude Fable 5 へのアクセスを停止する。他の Claude モデルは引き続き利用できる」と告知した。

プロダクト視点で重要なのは、これが単なるモデルの出し入れではなく、自分が日々回しているエージェントの中身が変わるという点だ。Claude のプロダクト全体で、新規セッションは選択中のモデル（実質 Opus 4.8 など）で走るようになった。Anthropic は影響緩和として 5 時間・週間のレートリミットを全ユーザーでリセットしている。先週 Fable 5 を前提に組まれていたワークフローは、いったん Opus 4.8 ベースで考え直す必要がある。

影響は Anthropic 外にも及んだ。Cognition は「Anthropic の発表と米政府指令を受け、Devin など自社プロダクトから Claude Fable 5 を外す。Opus 4.8 と GPT-5.5 を含む他モデルは引き続き利用可能」と即日アナウンス。Anthropic 自身が 6 月 14 日に SF で開催した Claude Build Day も、当初予定の Fable 5 ではなく Opus 4.8 上で実施された。

  • @ClaudeDevs: 全ユーザーの 5 時間・週間レートリミットをリセット
  • @cognition: Cognition 製品から Claude Fable 5 を外す
  • @ClaudeDevs: Build Day は Opus 4.8 上で実施

Claude Design と Claude Code が双方向同期——/design-sync でデザインシステムを行き来

6 月 17 日、Claude Design と Claude Code が双方向に同期するようになった。Claude Code 側で /design-sync を実行すると、自分のデザインシステムをリポジトリに取り込み、実際のコンポーネントに対してビルドできる。逆に Claude Code で作ったものを Claude Design に押し戻し、キャンバス上で編集を続けることもできる。デザインの受け渡しが「画像を見ながら作り直す」ではなく、実コンポーネント単位で往復する形になった。

Claude Design 自体もこのタイミングで強化された。リポジトリ・デザインファイル・コードベースからデザインシステムを取り込み、自分の実コンポーネントでビルドし、出力を見せる前にデザインシステムへの準拠を自己チェックする。キャンバス上でドラッグ・リサイズ・整列ができる新レイアウト操作や、PDF・PowerPoint へのエクスポートも加わった。Claude Design は全有料プラン（web・デスクトップ）でベータ提供されている。

  • @ClaudeDevs: Claude Code and Claude Design now sync both ways（/design-sync）
  • @claudeai: New in Claude Design — syncs with Claude Code
  • @claudeai: Claude Design is in beta on all paid plans

Managed Agents が本番運用へ——スケジュール実行・Vault 認証情報・各社サンドボックス対応

Claude Managed Agents が、実運用に必要なピースを相次いで埋めた。6 月 11 日にはスケジュール実行（タスクを定期実行）と、Vault の認証情報を環境変数として CLI に渡す環境変数クレデンシャルが追加された。Claude 自身はシークレットにアクセスせず、認証情報は安全に注入される設計で、環境変数で認証する CLI・SDK・直接 API 呼び出しをエージェントから安全に使えるようになる。

翌 12 日には、自分の管理下のサンドボックス（自前インフラや任意プロバイダ）でエージェントを動かすためのガイドが拡充され、Blaxel・E2B・Google Cloud・Namespace・Superserve に対応した。さらに 6 月 16 日には Applied AI チームが、認証情報・サンドボックス化・可観測性といった「エージェントを本番に乗せるときに必ずぶつかる課題」とその解き方をまとめたブログを公開している。先週ベータとして発表された要素が、実際に使える形へ整いつつある。

  • @ClaudeDevs: Scheduled deployments と環境変数クレデンシャルを追加
  • @ClaudeDevs: Blaxel・E2B・Google Cloud・Namespace・Superserve 向けサンドボックスガイド
  • @ClaudeDevs: Applied AI チームによる本番投入のブログ

Anthropic Economic Index——「Claude Code が実際に何に使われているか」をデータで公開

6 月 16 日、Anthropic が Claude Code の利用実態を追う経済インデックスのフレームワークを公開した。プライバシー保護分析で 2025 年 10 月〜2026 年 4 月の 40 万セッションを分類した結果、半分以上がコードの作成・修正、約 5 件に 1 件が OS・インフラ操作だった。タスクの金銭的価値はフリーランス市場での相場に換算して期間中に上昇しており、Claude Code 上の平均タスクが「より価値の高い仕事」へシフトしていることが示された。

使い手の専門性についても示唆がある。質問の仕方や語彙から判定したドメインエキスパートは成功率が高い一方、中級者と専門家の差は意外に小さく、「ある程度の習熟があれば十分」という結果になった。検証可能な完了証跡（コミットされたコードなど）を必須とする最も厳しい成功基準でも、各職種はソフトウェアエンジニアリングと 7 ポイント以内に収まっていた。プロダクトを「誰がどんな仕事に使っているか」を測る土台として、今後 Anthropic Economic Index に取り込まれていく。

  • @AnthropicAI: Claude Code をスケールで追う経済リサーチのフレームワーク
  • @AnthropicAI: 平均タスクの金銭的価値が上昇
  • @AnthropicAI: 検証可能な成功率は各職種がソフトウェアエンジニアと 7pt 以内

2.1.176〜2.1.181 を連続リリース——ripgrep 推奨・粒度の細かい権限ルール・/config key=value

Claude Code 本体は今週も高頻度でリリースが続いた。注目したい変更を拾うと、Grep が ripgrep ベースを推奨するようになり、コード検索がより速く一貫した挙動に（2.1.176／2.1.178）。権限ルールが Tool(param:value) 形式とワイルドカード * を受け付けるようになり、特定の引数値だけをブロックする細かい制御ができるようになった（2.1.178）。また availableModels がブロック済みモデルへ迂回する環境変数エイリアスを弾くようになっており、これは今回の Fable 5 停止のような「使えるモデルの差し替え」とも噛み合う変更だ。

安定性の修正も入っている。ストリーミングが途中で切れた際に部分応答を保持し、スピナーが "running tool" のまま固まる問題を解消（2.1.179）。サンドボックスの denyRead/allowRead グロブ処理がパスを過剰に膨らませないよう改善された。さらに 2.1.181 では /config key=value で任意設定を直接書き換えられるようになり、39 件の CLI 変更が入った大型リリースとなっている。リリースの一次情報は GitHub Releases（anthropics/claude-code）を参照したい。

  • @ClaudeCodeLog: 2.1.178 — ripgrep 推奨・Tool(param:value) 権限ルール
  • @ClaudeCodeLog: 2.1.179 — 部分応答の保持・サンドボックス glob 改善
  • @ClaudeCodeLog: 2.1.181 — /config key=value、39 件の CLI 変更
  • GitHub Releases: anthropics/claude-code

OpenAI Codex の動き

レートリミットのリセットを「貯めて後で使う」——Computer Use・プラグインマーケットも

Codex まわりは機能追加が活発だった。まず 6 月 12 日、レートリミットのリセットを自分の好きなタイミングで使えるよう貯蓄できる機能が Go/Plus/Pro/Business 向けにロールアウト（最初の 1 回は無料）。「みんなが自分のタイミングでリセットを使いたがるから出した」という告知文が国内エンジニアにも面白がられた。2 週間限定で、友人を招待すると双方にリセットが貯まる紹介キャンペーンも走っている。レートリミットの硬直性に対するユーザーの不満に、運用面で手当てした形だ。

機能面では Codex アプリに macOS / Windows での Computer Use、バックグラウンドのブラウザ操作向け Chrome 拡張、ワークフローや技術スタック・規約を覚える Memories が追加された。Codex CLI も 0.140.0 で /usage（日次・週次・累計のトークン統計）、Claude Code からの /import（設定・config・チャットの取り込み）、@ によるファイル・プラグイン・スキルの統合メンションを追加。0.141.0 ではリモート実行向けの E2E 暗号化 Noise リレーとプラグインマーケットプレイスが入った。Claude Code からの移行導線（/import）を公式に用意してきた点は、乗り換えを狙う動きとして注目しておきたい。

  • @OpenAI: レートリミットのリセットを貯めて後で使えるように
  • @Codex_Changelog: Codex アプリに Computer Use・Chrome 拡張・Memories
  • @Codex_Changelog: Codex CLI 0.140.0（/usage・Claude Code からの /import）
  • @Codex_Changelog: Codex CLI 0.141.0（暗号化リレー・プラグインマーケット）

OpenAI が Ona を買収——「ラップトップを閉じても走り続ける」実行基盤を Codex に

6 月 11 日、OpenAI が Ona（@ona_hq）の買収合意を発表した。同社のセキュアなクラウド実行技術により、Codex はラップトップを閉じていても長時間タスクを継続でき、より多くの組織が本番でエージェントを安全に運用できるようになるとしている。Codex の「アプリでの Computer Use」「暗号化リモートリレー」といった今週の追加と合わせて、ローカル端末に縛られないクラウド実行へ寄せていく方向性がはっきり見える。

その他プロダクトの動向

Grok Build がプラグインマーケットを開設——ターミナルから Vercel・Sentry・MongoDB 等を直叩き

xAI のコーディングエージェント Grok Build が、6 月 11 日にプラグインマーケットプレイスをベータ公開した。ターミナルから Vercel・Sentry・MongoDB・Cloudflare・Chrome DevTools のプラグインを使え、たとえば Sentry プラグインを入れればエージェントにエラーの発見・修正、スタックトレース解析、アラートのトリアージを頼める。Vercel プラグインなら本番デプロイやサンドボックス立ち上げ、Shadcn でのアプリ構築まで一プロンプトで通る。コーディングエージェント各社が「プラグイン／拡張マーケット」を競って整備する流れに、Grok も乗ってきた。

周辺では、SuperGrok / X Premium のサブスクで Warp 内から Grok Build モデルを使えるようになり（6/15）、Grok 4.3 が Amazon Bedrock 経由で利用可能に（6/17）。エージェント本体を各開発環境・クラウドに載せていく動きが続いている。

  • @xai: Grok Build Plugin Marketplace がベータ公開
  • @xai: Sentry プラグインでエラーの発見・修正・トリアージ
  • @xai: Warp 内から Grok Build モデルを利用可能に
  • @xai: Grok 4.3 が Amazon Bedrock で利用可能に

# 週間コーディングエージェントウォッチ

紙名: `コーディングエージェントウォッチ`
補助ラベル: `過去1週間`

## 調査依頼

過去 1 週間において、コーディングエージェントを中心とする開発ツール周辺で何が話題になり何が変わったかをまとめる。
リリースノートを全件網羅するのではなく、エンジニアが注目している動きを選んで紹介する。

## コア監視対象

### Claude Code

GitHub Releases（`https://github.com/anthropics/claude-code/releases`）を必ずチェックし、過去 1 週間の全リリースを取り上げる。
ただし、項目ごとに重要度を判定して並び順・分量を調整する。

## 関心領域（動的に拾う）

以下の領域で、x（旧 Twitter）や Hacker News などで言及・話題になっているものを動的に拾う。コア監視対象以外のツールは、明示的に話題になっているか重要な変更があったときのみ取り上げる。

- コーディングエージェント全般（Devin、Cursor、Cline、Aider、その他）
- コードレビュー・CI 連携ツール（CodeRabbit、その他）
- IDE 統合・開発体験を変えるツール
- 関連ライブラリ・フレームワーク

## 情報取得のルール

- x MCP を用いて、開発者コミュニティで話題になっている投稿・スレッドを参照する
- リリースノート（公式）を一次情報とし、x の言及量・反応で重要度を補強する
- スパム・宣伝色の強い投稿は除外する
- 取り上げる場合は、リリースノート URL、x の代表的な投稿 URL（任意）、公式ブログなどを出典に含める

## 出力構造

冒頭にまとめを置き、続いて Claude Code のセクション、その後に関心領域での注目動向を並べる。

- まとめ: 今週の最大の注目点（Claude Code を含むコーディングエージェント周辺全体で）を 3〜5 行で。冒頭に配置し、忙しいときはここだけ読めば全体を把握できるようにする
- Claude Code: 過去 1 週間の全リリースを取り上げる。重要なリリース・機能を上位に置き、軽微な変更は集約して短く触れる。各リリース見出しから対応するリリースノートにリンクする
- 関心領域での注目動向: コア監視対象以外で、x や Hacker News で言及されているプロダクト・機能・動向を、テーマ単位でセクション化する（例: 新しいコーディングエージェントの登場、既存ツールの大型アップデート、コミュニティで話題のワークフローなど）。各エントリに「なぜ注目されているか」を書き、出典 URL を末尾に置く

## 週次ならではの編集方針

- 「全リリース列挙」は避ける。Claude Code 以外のツールについては、x で話題になっているか重要な変更があったときのみ取り上げる
- 重要度は「公式の派手さ」ではなく「実際のユーザーの反応」で判断する
- 動向の見出しは「動向 1」「動向 2」のような番号ラベルにせず、その動向自体を示すタイトルにする

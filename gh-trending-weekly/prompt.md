# 週間 GitHub トレンド

紙名: `GitHub トレンド`
補助ラベル: `過去1週間`

## 調査依頼

過去 1 週間において GitHub でトレンドになったリポジトリ（全言語、Weekly ビュー）を調べ、読みやすい形に整理してまとめる。

## 情報取得のルール

- ソースは `https://github.com/trending?since=weekly`（全言語、週次）
- 各リポジトリについて、リポジトリ名（owner/repo）、概要、主な用途、付与スター数の傾向、リポジトリ URL を取得する
- 取り上げるのは上位 10〜15 件を目安に絞る（情報量が多すぎないようにする）
- 公式 README やリリースノート、関連記事も参照してよい

## 出力構造

冒頭にまとめを置き、続いて注目リポジトリを並べる。

- まとめ: 今週のトレンドの傾向・テーマ（例: AI 関連が多い、Rust 製ツールの台頭、特定領域への注目など）を 3〜5 行で。冒頭に配置し、忙しいときはここだけ読めば全体を把握できるようにする
- 注目リポジトリ: 上位 10〜15 件を、見出し（owner/repo と一言概要）と本文段落（何をするものか・なぜ今注目されているか）で列挙する。各エントリの末尾にリポジトリ URL を置く
- カテゴリでまとめられる場合は、テーマごとのセクション（例: AI／開発ツール／インフラ／ドキュメント など）に分けてもよい

## 週次ならではの編集方針

- 単純なスター順の羅列ではなく、「なぜトレンドに入ったか」のコンテキストを添える
- 似た領域のリポジトリが複数並ぶときは、同じセクションにまとめて比較できるようにしてよい
- 短命なリポジトリ（ジョーク、ミーム的なもの）は触れる程度に留め、本筋に時間を割く

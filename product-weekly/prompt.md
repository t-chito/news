# 週間プロダクトウォッチ

紙名: `プロダクトウォッチ`
補助ラベル: `過去1週間`

## 調査依頼

過去 1 週間において、自分が追う AI プロダクト（コーディングエージェントを中心とする）の動きをまとめる。
全件網羅ではなく、エンジニアが注目している動き・実際の使い勝手に関わる変化を選んで紹介する。

## 情報源

### x（一次素材）

`x-list-watch` スキルを使って、以下 2 つの list の過去 1 週間の投稿を取得する。

- Products list (`2052866057262846113`): プロダクト公式アカウント中心
- LLM list (`2052860422592799160`): 公式 + 中の人 + 個人エンジニア

両 list の投稿を入力にして、本紙の視点（自分が追うプロダクトとして何が変わったか・話題か）でフィルタする。

### 補強

- 主要プロダクトの公式リリースノート（Claude Code、Devin、CodeRabbit など）
- 公式ブログ・告知

## 視点とフィルタ

本紙は **プロダクト視点**。次のような投稿を拾う。

- プロダクトの新機能リリース・大型アップデート
- プロダクトの使い方・ベストプラクティスに関する個人エンジニアの発信
- バグ・破壊的変更の告知

逆に本紙の対象外（llm-weekly 側）:

- 投資・提携・規制などの業界マクロな話題
- 基盤モデル単体のリリース（プロダクトとしての話題でなければ）
- 研究・ベンチマーク

## 出力構造

冒頭にまとめを置き、続いて Claude Code のセクション、その後にその他プロダクトの動向を並べる。

- まとめ: 今週の最大の注目点を 3〜5 行で。冒頭に配置し、忙しいときはここだけ読めば全体を把握できるようにする
- Claude Code: 過去 1 週間の Claude Code 関連の動きを取り上げる。リリース、機能追加、中の人の発信、エンジニアの注目を集約。リリースは GitHub Releases（`https://github.com/anthropics/claude-code/releases`）を一次情報とする
- その他プロダクトの動向: Claude Code 以外で x で言及・話題になっているプロダクトを、テーマ単位でセクション化する（例: 新しいエージェント、既存ツールの大型アップデート、コミュニティで話題のワークフローなど）。各エントリに「なぜ注目されているか」を書き、出典 URL を末尾に置く

## 週次ならではの編集方針

- 「全リリース列挙」は避ける。Claude Code 以外のプロダクトは、x で話題になっているか重要な変更があったときのみ取り上げる
- 重要度は「公式の派手さ」ではなく「実際のユーザーの反応（x のエンゲージメント、個人エンジニアの言及）」で判断する
- 動向の見出しは「動向 1」「動向 2」のような番号ラベルにせず、その動向自体を示すタイトルにする

まとめ

今週のフロントエンドを一言で表すなら「速度と安全」。TypeScript 7.0 Beta が GoベースのコンパイラでTS 6.0比10倍速という数字を引っ提げて登場し、Angular 22 がSignal-First時代の本格幕開けを宣言した。一方、npm供給チェーン攻撃「Miasmaキャンペーン」が @redhat-cloud-services スコープの32パッケージを汚染し、セキュリティ意識を引き締める週となった。Node.js のセキュリティリリースも翌17日に控えており、アップデート対応を優先したい。

今週の重要ポイント

TypeScript 7.0 Beta — GoコンパイラでTS史上最大の性能革命

4月21日に公開されたTypeScript 7.0 Betaは、コンパイラをGoへ完全移植した初のベータ版。型チェックの意味論はTS 6.0と同一を保ちつつ、ベンチマークでは約10倍の高速化を達成した。安定版リリースは6月末から7月初旬が見込まれている。

現時点では es2021 以降のターゲットのみサポートし、デコレータやウォッチモードはまだ未対応。VS Code拡張との本格統合は2026年中ごろを予定。

  • TypeScript 7.0 Beta Arrives on Go-Based Foundation With 10x Speed Claim — Visual Studio Magazine
  • TypeScript 7.0 Beta Is Out — The Compiler Is Now Written in Go and It's 10x Faster

Angular 22 正式リリース — Signal-Firstアーキテクチャが本番稼働へ

6月3日にリリースされたAngular 22は、Signal Forms・Zoneless・OnPushがすべて本番安定版に昇格し、「Signal-First」時代の実質的な出発点となった。セレクタレスコンポーネントやdebounced()シグナルなど使い勝手を高める機能も追加。

CLIにはAIアシスタントと連携する実験的MCPサーバーが同梱され、コード生成・マイグレーション支援・ドキュメント検索を自動化できる。

  • Angular 22: The Release That Finally Makes Angular Feel 10 Years Ahead — Medium
  • Angular 22: The "Signal-First" Era Is Finally Here — Medium

npm供給チェーン攻撃「Miasma」— インストール時にマルウェアが自動実行

6月1日、Wiz Research が @redhat-cloud-services スコープの32パッケージへの不正改ざんを報告。npm install 実行と同時に4.2 MBの難読化ペイロードが走る仕組みで、認証情報を窃取するワーム型マルウェア「Miasma」が確認されている。

該当パッケージを利用している場合は即時バージョン確認と依存ツリーの監査を推奨。

  • Miasma: Supply Chain Attack Targeting RedHat npm Packages — Wiz Blog
  • Axios npm Hijack 2026: Everything You Need to Know — SOCRadar

Svelte June 2026 — TypeScript 6.0 サポートと SvelteKit の破壊的変更

Svelte Language ToolsがTypeScript 6.0に対応し、language server・svelte2tsx・svelte-checkの各パッケージが更新された。SvelteKit ではリモートフォームのバリデーション漏れを検出する警告が追加された。

破壊的変更として、リモートクエリの .run() が廃止。すべてのコンテキストで await query() を直接使用する形に移行が必要。

  • What's new in Svelte: June 2026 — svelte.dev

Node.js セキュリティリリース — 6月17日（火）公開予定

Node.js プロジェクトは6月17日にセキュリティリリースを予告している。詳細は公開後に明らかになるが、CVE情報の確認とアップデート対応は速やかに行いたい。

  • Node.js — Wednesday, June 17, 2026 Security Releases

言語・コンパイラ

TypeScript 6.0 → 7.0 への移行パス

3月23日にリリースされたTypeScript 6.0は、JavaScriptベースの最終版として位置づけられ、多数の非推奨フラグと削除が含まれる。7.0への移行を意識したコードベース整理を今から進めておくことが推奨される。

TS 6.0→7.0では意味論が保たれるため、型チェックの挙動変化は最小限の見込みだが、古いコンパイルターゲット（ES2020以下）は7.0では当面非サポートになる点に注意。

  • Announcing TypeScript 6.0 — TypeScript Blog

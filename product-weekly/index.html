まとめ

今週の主役は Claude Tag（6/23 発表）。Anthropic は「Claude Code をマルチプレイヤー・非同期・能動的にしたもの」と位置づけ、Slack のチャンネルに Claude を 1 人のチームメンバーとして常駐させる。鍵は エージェントアイデンティティ——従来の「ユーザーになりすまして動く」のではなく、Claude が自分自身の GitHub・Linear・サービスアカウントを持って動く新しいアクセスモデルだ。Enterprise / Team プラン向けにベータ提供で、Anthropic 社内では「プロダクトチームのコードの 65% を Claude が書いている」と公表した。Claude Code 本体も 2.1.186〜2.1.191 を連続リリースし、/rewind・claude mcp login・sandbox.credentials など実務寄りの機能が入った。他陣営では、Notion が Claude と Cursor を共有タスクボードの「同僚」として置く External Agents を投入、Devin が PR レビュー前の自動 QA と Kimi K2.7 / GLM 5.2 の無料開放を実施、OpenAI は自社初の AI チップ Jalapeño と GPT-5.5 Instant の刷新を出した。エージェントを「個人の道具」から「チームの一員」へ押し上げる動きが各社で同時に進んでいる。

Claude Code

Claude Tag 登場——Claude Code が「マルチプレイヤー・非同期」になり、Slack のチームに常駐する

6 月 23 日、Anthropic は Claude Tag を発表した。Slack のチャンネルに Claude が参加し、選んだチャンネルとツールにアクセスできる「チームメンバー」として働く。スレッドで Claude をメンションしてタスクを渡せば、Claude が作業を段階に分解し、与えられたツールを使って進め、スレッドに成果を返す。PR の作成・マージ、データ分析、インシデント対応まで担う。Anthropic 自身は「Claude Tag は Claude Code の進化形。より能動的で、チーム全体で使えるように作られている」と説明し、「Claude Code は今もソロの同期作業には最速。Claude Tag はそれをマルチプレイヤー・非同期・能動的にしたもの」と両者を整理している。

プロダクト視点で重要なのは、これが「個人の手元で回すエージェント」から「チームに常駐するエージェント」への転換である点だ。チャンネルには全員と対話する 1 つの Claude が居り、チャンネルを追いながら文脈を蓄積するので、同僚が途中から引き継いでも一から説明し直す必要がない。アンビエント動作をオンにすれば、静まったスレッドを能動的にフォローアップし、関連する変化を横断的に拾い上げる。Anthropic は具体的なユースケースとして、インシデント対応（ページ着信→グラフ取得・デプロイ差分・根本原因の特定→修正 PR→メトリクス復旧の確認）、バグトリアージ（フィードバックチャンネルから自動でレポートを拾い、再現・git blame・修正・所有者タグ付け）、A/B テストのガードレール監視、ブロックされた依存作業の待ち受けなどを挙げている。

注目を集めたのが、その裏側にある 「エージェントアイデンティティ（agent identity）」 という考え方だ。複数人が居るチャンネルで Claude が動くとき、誰の権限を借りるべきか——という問いに対し、Claude Tag は「Claude は自分自身として振る舞う」と答える。チャンネルでは Claude が自分の Linear アカウント・GitHub アカウント・データウェアハウスのサービスアカウントを持ち、各システムのログに Claude 名義で行動が記録される。監査すべき ID は 1 つで、一度失効させれば全連携でアクセスが切れる。一方 DM では従来どおり「あなたとして」動く。管理者がチャンネルごとに到達範囲をスコープできる設計だ。国内エンジニアからも「個人アカウント連携ではなくチャンネルに紐付いた 1 つの bot を従量課金でみんなで使う形」「スレッドでメンションしなくても会話が続けられるのが歓喜」といった反応が出ており、従来の Slack 連携から一段進んだ体験になっている。

Claude Tag は Claude Enterprise / Team プラン向けに Slack 上でベータ提供が始まった。Anthropic は「Claude Code チームは今年ずっと社内で Claude Tag を使って開発してきた。今ではプロダクトチームのコードの 65%——Claude Tag 自体を作ったコードの大半を含む——を Claude が書いている」としており、自社のドッグフーディングの規模感も示した。

  • @claudeai: Introducing Claude Tag — Slack でチームメンバーとして働く Claude
  • @claudeai: Claude Tag は Claude Code の進化形、社内コードの 65% を生成
  • @ClaudeDevs: Claude Code はソロ同期、Claude Tag はマルチプレイヤー・非同期・能動的
  • @ClaudeDevs: 「エージェントアイデンティティ」——Claude が自分自身の認証情報で動く
  • @claudeai: Enterprise / Team プラン向けに Slack でベータ提供
  • @yoshiko_pg: スレッドでメンションしなくても会話が続けられる
  • @oikon48: 「エージェントアイデンティティ」という概念が登場しているのが面白い

2.1.186〜2.1.191 を連続リリース——/rewind・claude mcp login・sandbox.credentials

Claude Code 本体は今週も高頻度でリリースが続いた。実務に効く変更を拾うと、まず 2.1.186 で claude mcp login <name> / claude mcp logout <name> が追加され、インタラクティブな /mcp メニューを開かずに CLI から MCP サーバーを認証できるようになった（--no-browser で SSH 越しの認証にも対応）。! から始まるシェルコマンドの出力に対して Claude が自動で応答するようになり（respondToBashCommands で従来挙動に戻せる）、/plugin に Skills セクションも加わった。

2.1.187 はセキュリティと組織運用まわりが中心。sandbox.credentials 設定で、サンドボックス内のコマンドが認証情報ファイルやシークレット環境変数を読み取れないようにブロックできるようになった。さらに 組織が設定したモデル制限 がモデルピッカー・--model・/model・ANTHROPIC_MODEL すべてに反映され、制限されたモデルを選ぶと「組織の設定で制限されている」と表示される。リモート MCP のツール呼び出しが 5 分でアボートするようになり、無限ハングも解消した。2.1.191 では /rewind が入り、/clear を実行する前の会話状態から再開して文脈を復元できるように。タスクパネルから止めたバックグラウンドエージェントが復活しなくなり「停止」が確実になった。リリースの一次情報は GitHub Releases（anthropics/claude-code）を参照したい。

  • @ClaudeCodeLog: 2.1.186 — claude mcp login/logout、! コマンドの自動応答
  • @ClaudeCodeLog: 2.1.187 — sandbox.credentials・組織のモデル制限・リモート MCP 5 分アボート
  • @ClaudeCodeLog: 2.1.191 — /rewind、停止したバックグラウンドエージェントが復活しない
  • GitHub Releases: anthropics/claude-code

「エージェントが同僚になる」——各社が共有ワークスペースへ

Notion External Agents——Claude と Cursor を共有タスクボードの「同僚」として置く

6 月 24 日、Notion が External Agents を発表し、Claude と Cursor を Notion 内に外部エージェントとして招けるようにした。チームで共有するボードからタスクを割り当て、同僚のように @ メンションして走らせ、実行の様子を見られる。Claude エージェントはデータ分析・コーディング・スプレッドシートやスライド作成など用途別に作って共有ボードに並べ、ワークフローとして連鎖させたり互いにタスクを引き継いだりできる。Cursor 側はバグ修正・機能開発・コードレビューといった開発ライフサイクル向けで、割り当てると Cursor の Cloud Agent がコードベースを読んで PR を開き、その実行は Notion 上で可視・レビュー可能なまま残る。Notion で始めた作業を Cursor 側で続ける往復もできる。

Claude Tag が「Slack に Claude が常駐する」方向なら、Notion は「自分のワークスペースに各社のエージェントを同僚として迎える」方向だ。Anthropic・OpenAI（Codex）・Cursor といった競合のエージェントを横並びでボードに載せる立場を取った点が、プラットフォームとしての Notion らしい動きと言える。

  • @NotionHQ: Introducing External Agents in Notion — Claude + Cursor
  • @NotionHQ: Meet Claude agents in Notion
  • @NotionHQ: Meet Cursor in Notion — タスクボードの新しい同僚

Devin が「レビュー前に自分でテスト」を実装、Kimi K2.7 / GLM 5.2 も無料開放

Cognition の Devin に、PR をレビューに回す前に自分で動作確認する機能が入った。Devin がテスト計画を立て、ユーザーがそれを承認すると、ステップごとの QA を視覚的なチェックリスト付きの画面録画で返す。「エージェントが書いて投げっぱなし」ではなく、人間のレビュー前に検証証跡を用意する方向で、レビュー負荷を下げにいく動きだ。

あわせて Devin Desktop / CLI で Kimi K2.7 Code と GLM 5.2 が利用可能になり、Pro / Max / Teams ユーザーは 7 月 5 日まで両モデルを無料で試せる。Cognition 独自の実務ベンチ FrontierCode Extended では GLM 5.2 が 43.0%、Kimi K2.7 Code が 39.5% で、GPT-5.5（44.8%）に近く Claude Opus 4.8（51.8%）を追う「競合ティア」に位置づけられている。オープンウェイト勢を実コーディングエージェントから無料で触らせ、ユーザーに乗り換え候補として体験させる狙いが見える。

  • @cognition: Devin はレビュー前に自分でテストし、画面録画の QA チェックリストを返す
  • @devindesktop: Kimi K2.7 Code と GLM 5.2 が Devin Desktop / CLI で利用可能（7/5 まで無料）
  • @devindesktop: FrontierCode Extended のスコア（GLM 43.0% / Kimi 39.5% / GPT-5.5 44.8% / Opus 4.8 51.8%）

その他プロダクトの動向

OpenAI が自社初の AI チップ Jalapeño を発表、GPT-5.5 Instant と Codex CLI も更新

6 月 24 日、OpenAI が自社で設計・製造した初の AI チップ Jalapeño を発表した。Broadcom と組んで量産まで持ち込んだ LLM 推論専用チップで、ChatGPT・Codex・API・今後のエージェント製品のワークロードを支える。Greg Brockman は「9 か月かけて推論用にゼロから設計し、自社モデルで設計を加速した。ワットあたり性能が素晴らしい」とアピールした。プロダクトの裏側を支える基盤として、Codex を含むエージェント群の供給能力・コストに効いてくる話だ。

同日、最も使われているモデル GPT-5.5 Instant の新バージョンも投入された。質問の意図をより正確に汲み、複雑な制約も安定して扱えるようになったという。Codex CLI も 0.142.0 で /usage のクレジット引き換え（リトライ付き）・/plugins のセクション分け（Curated / Workspace / Shared）・サーバー承認 URL 付きのインデックス済み Web 検索を追加、0.142.1 では Windows のシステムプロキシ認証対応（PAC / WPAD 自動検出）を入れている。

  • @OpenAI: 自社初の AI チップ Jalapeño を発表
  • @gdb: Jalapeño は LLM 推論向けに 9 か月かけてゼロから設計
  • @OpenAI: GPT-5.5 Instant の新バージョン
  • @Codex_Changelog: Codex CLI 0.142.0（/usage・/plugins セクション・インデックス済み Web 検索）

Grok Build のプラグインマーケットが拡充——MongoDB・Firecrawl が追加

先週ベータ公開された xAI の Grok Build プラグインマーケットプレイスに、新しいプラグインが加わった。公式 MongoDB プラグインでデータのクエリ・インデックス最適化・DB 管理をターミナルから行え、Firecrawl プラグインで Web スクレイピングを組み込める。コーディングエージェント各社が「拡張・プラグインのマーケット」を競って育てる流れの中で、Grok も対応サービスを着実に増やしている。

  • @xai: Grok Build の公式 MongoDB プラグイン
  • @xai: Grok Build Plugin Marketplace に Firecrawl プラグイン

コミュニティから

「ループエンジニアリング」と、持続可能な AI モブプロの模索

エージェント前提の開発スタイルを言語化する動きが続いている。@t_wada は「ループエンジニアリングはサイバネティクスの再発見だ」という記事を取り上げ、流行語化したこの概念の構造を 1 年以上前から言い当てていた、と評された投稿を共有。別の投稿では「AI 疲れ軽減のためにモブプロを始めたら抜群の成果と引き換えに人間が疲弊したので、持続可能なモブプロをガチで模索してみた」という実践事例を紹介し、AI モブプログラミングの「成果と疲労のトレードオフ」に注目が集まった。エージェントを使い倒す段階に入ったからこそ、人間側の運用設計が論点になってきている。

モデルの使い分けに関する現場感も。@yoshiko_pg は「Opus 4.8 はスキル追従こそ弱いが、コーディング強化のおかげで pptx を作るのは上手い。コーディングとビジュアル両方の強さが要る」と、用途ごとのモデルの当たり外れを実感ベースで共有している。

  • @t_wada: 持続可能なモブプロをガチで模索してみた
  • @yoshiko_pg: Opus 4.8 はスキル追従は弱いが pptx は上手い

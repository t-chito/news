# 週間フロントエンドニュース

紙名: `フロントエンドニュース`
補助ラベル: `過去1週間`

## 調査依頼

過去 1 週間における web フロントエンド関連のトピックを調べ、週単位の動向として読み取れる形に整理してまとめる。

## トピック

- 主なニュース（フレームワーク、言語、トレンドなどのカテゴリ別に）
- プロバイダやコミュニティからのアナウンス（例: React 公式のリリース）
- 関連するツールやライブラリのアップデート

## 情報取得のルール

- 情報は公式のブログ、サイト、一般に信頼できると言えるような有名なソースから取得する
- アナウンス日時がわかれば記載する

## 出力構造

冒頭にまとめを置き、続いて「今週の重要ポイント」、その後にテーマ別動向、最後に「来週の注目ポイント」を並べる。

- まとめ: 今週の最大の注目点を 3〜5 行で。冒頭に配置し、忙しいときはここだけ読めば全体を把握できるようにする
- 今週の重要ポイント: 3〜5 件の重要トピックを、見出しと 2〜3 行の要約で列挙する
- テーマ別動向: カテゴリ単位（例: セキュリティ、フレームワーク・ランタイム、エコシステム・ツール、ライブラリ、デザイン・UI など）のセクションを並べる。各テーマ内に動向セクション（記事）を 1 つ以上置く。各動向の末尾にその動向に対応する出典 URL のリストを置く
- 来週の注目ポイント: 翌週に予定されているイベント、発表予定など（情報があれば）

## 週次ならではの編集方針

- 同じテーマ・プロジェクトをめぐる週内の複数の動きがストーリーとしてつながるなら、1 つの動向にまとめてよい
- ストーリーとしてつながらないが重要な話題は、無理に統合せず独立した動向として並列に置く
- 日次の速報的な羅列・似た情報の重複は避ける。軽微な進捗は最重要点の中に集約する
- 動向の見出しは「動向 1」「動向 2」のような番号ラベルにせず、その動向自体を示すタイトルにする

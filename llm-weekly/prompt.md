# 週間 LLM ニュース

紙名: `LLM ニュース`
補助ラベル: `過去1週間`

## 調査依頼

過去 1 週間における LLM 業界全体のマクロな動向を、週単位の流れとして読み取れる形に整理してまとめる。

## 情報源

### x（一次素材）

`x-list-watch` スキルを使って、以下 2 つの list の過去 1 週間の投稿を取得する。

- Products list (`2052866057262846113`): プロダクト公式アカウント中心
- LLM list (`2052860422592799160`): 公式 + CEO・研究者・中の人・個人エンジニア

両 list の投稿を入力にして、本紙の視点（LLM 業界全体としてどんな動きか）でフィルタする。

### 補強

- 公式ブログ・告知（プロバイダ、研究機関、政府・規制機関）
- 信頼できるテック媒体（TechCrunch、CNBC、InfoQ など）

## 視点とフィルタ

本紙は **業界マクロ視点**。次のような投稿を拾う。

- 主要プロバイダの基盤モデル・大型機能のリリース
- 推論インフラ・ランタイムのアップデート（クラウド事業者、専用チップ、最適化など）
- LLM 関連のリサーチ・ベンチマーク
- 業界動向（投資、提携、規制、上場、組織変化）
- CEO・研究者・中の人の戦略的な発言

product-weekly と重なる話題（プロダクトのリリース等）は、本紙では「業界としてどんな意味があるか」の角度で扱う。プロダクト視点の使い勝手・機能の細部は product-weekly 側。

## 出力構造

冒頭にまとめを置き、続いて「今週の重要ポイント」、その後にテーマ別動向、最後に「来週の注目ポイント」を並べる。

- まとめ: 今週の最大の注目点を 3〜5 行で。冒頭に配置し、忙しいときはここだけ読めば全体を把握できるようにする
- 今週の重要ポイント: 3〜5 件の重要トピックを、見出しと 2〜3 行の要約で列挙する
- テーマ別動向: カテゴリ単位（例: モデルリリース、推論インフラ、エージェント・ツール、リサーチ・ベンチマーク、業界動向 など）のセクションを並べる。各テーマ内に動向セクション（記事）を 1 つ以上置く。各動向の末尾に対応する出典 URL のリストを置く
- 来週の注目ポイント: 翌週に予定されているイベント、発表予定など（情報があれば）

## 週次ならではの編集方針

- 同じプロバイダ・モデルをめぐる週内の複数の動きがストーリーとしてつながるなら、1 つの動向にまとめてよい
- ストーリーとしてつながらないが重要な話題は、無理に統合せず独立した動向として並列に置く
- 日次の速報的な羅列・似た情報の重複は避ける。軽微な進捗は最重要点の中に集約する
- 動向の見出しは「動向 1」「動向 2」のような番号ラベルにせず、その動向自体を示すタイトルにする

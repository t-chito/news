まとめ

今週も「AI エージェント向けスキル集」が週間トレンドの中心だが、潮目が変わりつつある。先週まで個別スキルの単発リポジトリが上位を占めていたのに対し、今週は addyosmani/agent-skills（週間 +9,285★）や PM 特化の phuryn/pm-skills といった「スキルをまとめたマーケットプレイス／フレームワーク」型が台頭した。同時に、急増するスキルの安全性を検査する NVIDIA/SkillSpector が新規にランクインし、エコシステムが「作る」段階から「使いこなす・守る」段階へ移ったことを示している。エージェントの実行ログ分析（agentsview）やコードベースの知識グラフ化（codebase-memory-mcp）など、エージェント運用を支える周辺ツールも厚みを増した。LLM のトークン／推論コスト最適化（headroom、LMCache）への関心は継続。AI 以外では Apple の container が v1.0.0 に到達したほか、iptv-org/iptv や chatwoot といった定番 OSS も安定して新規スターを集めている。

AI エージェントスキル集 & セキュリティ

addyosmani/agent-skills — 開発ライフサイクルを通すスキル集（週間 +9,285★ / 累計 6.3万★）

Google の Addy Osmani 氏による、AI コーディングエージェント向けの「実務級エンジニアリングスキル」集。要件定義から出荷までを /spec・/plan・/build・/test・/review・/code-simplify・/ship の 7 コマンドにマッピングし、計 24 のスキルを Define／Plan／Build／Verify／Review／Ship の 6 段階に整理している。

狙いは、エージェントが「最短経路」で雑に実装するのを防ぎ、仕様・テスト・セキュリティレビュー・品質ゲートをシニアエンジニア並みに踏ませること。コードレビュアー・テストエンジニア・セキュリティ監査・Web パフォーマンス監査の 4 つの専門ペルソナと 5 つの参照チェックリストも付属する。Claude Code・Cursor・Antigravity に対応。単発スキルから「開発プロセスごと型に流し込む」方向への移行を象徴する存在だ。

  • addyosmani/agent-skills

phuryn/pm-skills — プロダクトマネジメント特化のスキルマーケット（週間 +3,663★）

プロダクトマネージャー向けに、ディスカバリー・戦略・実行・ローンチ・グロースをカバーするスキル群。9 プラグインに 68 スキル／42 ワークフローを収め、アイデア検証や機会マッピング、PRD・OKR・ロードマップ作成、ペルソナ・市場規模推計、SQL 生成やコホート分析・A/B テスト、Go-to-Market のビーチヘッド選定まで、確立された PM フレームワークをそのままワークフロー化している。

Claude Cowork（推奨）・Claude Code CLI・Codex CLI から導入できる。エンジニア向けのスキル集が定着したのに続き、非エンジニア職種へとスキルエコシステムが横展開している好例だ。

  • phuryn/pm-skills

NVIDIA/SkillSpector — エージェントスキルの脆弱性スキャナー（週間 +5,505★ / 新規）

NVIDIA が公開した、AI エージェントスキルをインストール前に検査するセキュリティスキャナー。README によれば「スキルの 26.1% に脆弱性があり、5.2% は悪意がある可能性が高い」とされ、急増するサードパーティスキルの導入前評価を埋める位置づけだ。

Git リポジトリ・URL・ZIP・ディレクトリ・単一ファイルを入力に取り、プロンプトインジェクション・データ持ち出し・権限昇格・サプライチェーン攻撃など 16 カテゴリ 64 パターンを検査する。高速な静的解析に加えて任意で LLM による意味解析を行う二段階パイプライン、OSV.dev への CVE ライブ照会（オフライン時はフォールバック）、0〜100 のリスクスコアと導入可否の提示、Terminal／JSON／Markdown／SARIF 出力に対応する。スキルを「作る」競争の裏で「守る」需要が顕在化したことを示す象徴的な新顔だ。

  • NVIDIA/SkillSpector

エージェント運用・コード理解ツール

kenn-io/agentsview — コーディングエージェントのログ・コストを横断分析（週間 +1,382★）

Claude Code・GitHub Copilot・Cursor など 40 以上のコーディングエージェントのセッションを自動で発見・索引化し、ローカル完結で分析できるツール。「アカウント不要、すべてローカル、バイナリ 1 つ」を掲げ、SQLite FTS5 による全文検索、LiteLLM 連携のトークン・コスト追跡、アクティビティヒートマップやツール利用・ベロシティのダッシュボードを提供する。

Server-Sent Events によるライブ更新、SQLite（標準）／PostgreSQL（チーム共有）／DuckDB（ミラー）のマルチバックエンド、HTML や GitHub Gist へのセッション書き出しにも対応。Go 製で、複数エージェントを併用する開発者が「どこにいくら使ったか」を一元把握する用途に向く。

  • kenn-io/agentsview

DeusData/codebase-memory-mcp — コードベースを知識グラフ化する MCP サーバー（週間 +3,244★）

ソースコードから永続的な知識グラフを構築し、エージェントがファイルを逐次検索せずに構造を把握できるようにする MCP サーバー。平均的なリポジトリをミリ秒で、Linux カーネル（2,800 万行）でも 3 分で全索引化すると謳う。tree-sitter 文法を単一バイナリに同梱して 158 言語に対応し、Python・TypeScript・Go・Rust・Java・C# など主要言語ではハイブリッド LSP による型解決も行う。

検索・呼び出し追跡・影響分析・Cypher クエリ・デッドコード検出・HTTP ルート連携など 14 の MCP ツールを備え、ファイル単位の探索に比べてトークンを 99% 削減できるとする。依存ゼロの静的バイナリ、バックグラウンド差分同期、3D グラフ可視化（localhost:9749）も。Claude Code を含む 11 のエージェントを自動設定する。

  • DeusData/codebase-memory-mcp

Panniantong/Agent-Reach — API 不要でネット全体にアクセス（週間 +7,856★）

Twitter/X・Reddit・YouTube・GitHub・Bilibili・小紅書（XiaoHongShu）など 10 以上のプラットフォームの読み取り・検索能力をエージェントに付与する CLI。有料 API を使わずオープンソースバックエンドと無料エンドポイントを組み合わせる設計で、先週に続き週間 7,000 超のスターを維持し、累計 3.4 万に達した。エージェントに「外の世界を見る目」を安価に与えるニーズが根強いことを示している。

  • Panniantong/Agent-Reach

LLM 推論・トークンコスト最適化

chopratejas/headroom — トークン削減ツール、出力削減と失敗学習を追加（週間 +10,159★）

LLM に到達する前にツール出力・ログ・RAG チャンク・ファイル・会話履歴を圧縮し、回答品質を保ったままトークンを 60〜95% 削減するコンテキスト圧縮レイヤー。先週に続きトップ級の伸びを見せ、今週は v0.26.0 までバージョンが進んだ。

JSON（SmartCrusher）・コード AST（CodeCompressor）・散文（Kompress-v2 モデル）といったコンテンツ別の圧縮に加え、新たに出力トークンの削減（モデルの冗長さを steering と effort ルーティングで抑制）や、失敗学習（headroom learn が失敗セッションを掘り起こして補正を自動記述）といった機能が前面に出てきた。ライブラリ・プロキシ・MCP サーバー・エージェントラッパーの 4 形態で導入でき、可逆圧縮（CCR）で元データを後から復元できる点も維持。入力削減一辺倒から、入出力・運用全体の効率化へと射程を広げている。

  • chopratejas/headroom

LMCache/LMCache — LLM 推論を高速化する KV キャッシュ層（週間 +825★）

一時的な KV キャッシュを再利用可能な永続データに変え、複数のサービングエンジン・ストレージ間で保存・再利用・監視できるようにする KV キャッシュ管理レイヤー。長文コンテキストのエージェントワークフローや多ターン対話、RAG など知識拡張系を主対象とし、prefill 計算の重複を排除して初回トークンまでの時間（TTFT）短縮とスループット向上を図る。

エンジン非依存のデーモンとして動作し、GPU／CPU／ローカル／リモートへの階層的オフロード、Redis・S3 などプラガブルなバックエンド、prefix を超えた非接頭辞 KV 再利用、PD ディスアグリゲーションに対応。vLLM 連携を軸に、推論基盤レイヤーでの効率化が引き続き注目されている。

  • LMCache/LMCache

定番 OSS・インフラ

apple/container — Mac 向け Linux コンテナツールが v1.0.0 に到達（週間 +7,671★）

Apple 公式の、Mac 上で軽量仮想マシンとして Linux コンテナを動かすツール。Swift 製で Apple Silicon に最適化され、OCI 互換イメージのビルド・実行・レジストリへの push に対応する。先週まで 1.0 前のプレリリースだったが、今週 v1.0.0 に到達し、パッチバージョン内での安定性が保証される段階に入った。低レベル処理は Apple の Containerization Swift パッケージに依存。動作には新しい仮想化・ネットワーク機能を使うため macOS 26 以上が必要で、Docker Desktop の代替として Mac 開発者の支持を集め続けている。

  • apple/container

iptv-org/iptv — 世界中の公開 IPTV チャンネル集（週間 +7,899★ / 累計 12.5万★）

世界中で公開されている IPTV（インターネットテレビ）チャンネルを集めたプレイリスト集。M3U 形式に対応するプレイヤーに index.m3u を読み込ませるだけで多数のライブ配信を視聴できる。用途別の派生プレイリスト、番組表（EPG）ユーティリティ、プログラムからアクセスする API も整備されている。コミュニティ主導の長寿プロジェクトで、Unlicense（パブリックドメイン）の手軽さもあって今週も大量の新規スターを集めた。

  • iptv-org/iptv

chatwoot/chatwoot — Intercom 代替のオープンソース顧客サポート基盤（週間 +2,422★）

Intercom・Zendesk・Salesforce Service Cloud の代替を狙う、セルフホスト可能なオープンソース顧客サポートプラットフォーム。ライブチャット・メール・Facebook・Instagram・WhatsApp・Telegram・LINE・SMS などを 1 つの受信箱に集約し、定型問い合わせを自動化する Captain AI エージェント、ヘルプセンター、ラベルや定型文・自動割り当てなどのコラボ機能、CSAT を含むレポートを備える。Rails + Vue.js 製で、商用 SaaS を避けたい組織からの定番として安定した支持を保っている。

  • chatwoot/chatwoot
